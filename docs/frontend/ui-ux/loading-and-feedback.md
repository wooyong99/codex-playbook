# Loading & Feedback

로딩 UX는 모든 페이지에 걸쳐 있으며 기준이 없으면 화면마다 제각각이 된다.
스켈레톤 UI, 로딩 상태, 토스트, 성공/실패 메시지에 대한 기준을 이 문서에서 통합 관리한다.

---

## 로딩 처리 기준

### 초기 로딩은 Skeleton 사용

페이지에 처음 진입하거나 주요 데이터를 처음 불러올 때는 스켈레톤 UI를 사용한다.

- 화면 레이아웃을 먼저 보여줌으로써 레이아웃 시프트(Layout Shift)를 방지
- 스켈레톤은 실제 콘텐츠의 형태(텍스트 줄, 카드, 테이블 행)를 반영
- 단순 회색 박스 나열이 아닌 구조를 갖춘 스켈레톤 사용

```
적용 대상:
- 페이지 첫 진입 시 목록/테이블 로딩
- 상세 페이지 데이터 로딩
- 대시보드 지표 카드 로딩
```

### 짧은 액션 로딩은 버튼 로딩으로 충분

저장, 삭제, 상태 변경 등 버튼 클릭으로 시작되는 액션은 버튼 자체에 로딩 상태를 표시한다. 별도의 스켈레톤이나 스피너가 필요하지 않다.

```
[저장] → [● 저장 중...] → [저장]
```

- 버튼 로딩 중 disabled 처리로 중복 제출 방지
- 응답 완료 후 원래 버튼 상태로 복귀

### 전체 페이지 블로킹 로딩은 최소화

전체 화면을 가리는 로딩 오버레이는 사용자 인터랙션을 완전히 차단하므로 최소화한다.

| 허용 | 비허용 |
|------|--------|
| 파일 대량 업로드 | 단순 목록 조회 |
| 긴 작업의 진행 상태 표시 | 저장/수정 요청 |
| 권한 확인 등 필수 초기화 | 페이지 이동 |

전체 블로킹이 불가피한 경우에도 진행률이나 진행 메시지를 표시한다.

### 테이블 로딩은 Row Skeleton 우선

테이블 데이터 로딩 시 빈 화면 대신 행(row) 형태의 스켈레톤을 표시한다.

```
| ████████ | ████ | ██████ |   ← skeleton row
| ████████ | ████ | ██████ |
| ████████ | ████ | ██████ |
```

- 스켈레톤 행 수는 기본 페이지 크기(예: 10개)와 동일하게 설정
- 필터 또는 페이지 변경 시에도 동일한 row skeleton 방식 적용
- 테이블 헤더(컬럼명)는 로딩 중에도 유지하여 레이아웃 안정성 확보

## 피드백 처리 기준

### 저장 성공 시 Toast 표시

저장, 수정, 생성 액션이 성공하면 토스트 메시지로 즉시 피드백을 제공한다.

| 항목 | 기준 |
|------|------|
| 색상 | green 계열 |
| 위치 | 화면 우측 상단 또는 우측 하단 (전체 프로젝트 통일) |
| 자동 닫힘 | 3초 후 |
| 문구 형식 | "{대상}이 저장되었습니다." |

```
✓ 카테고리가 저장되었습니다.
```

페이지 이동이 발생하는 경우(저장 후 목록 페이지로 이동) 토스트는 이동 후 화면에서 표시한다.

### 삭제 성공 시 목록 반영 방식

삭제 성공 후 처리 순서:

1. 토스트 메시지 표시 (`✓ 삭제되었습니다.`)
2. 목록 쿼리 invalidation → 자동 재조회
3. 현재 페이지에 항목이 없어지면 이전 페이지로 이동

삭제된 항목을 로컬 상태에서 직접 제거하지 않는다. 서버 재조회를 통해 목록을 갱신한다. (단, 즉각적인 UX가 필요한 경우 Optimistic Update 허용 — [state-management.md](../architecture/state-management.md) 참고)

### 실패 시 사용자 액션 유도 문구 제공

오류 발생 시 기술적 오류 코드가 아닌, 사용자가 취할 수 있는 행동을 안내하는 문구를 제공한다.

| 상황 | 금지 문구 | 권장 문구 |
|------|-----------|-----------|
| 네트워크 오류 | "500 Internal Server Error" | "요청을 처리하지 못했습니다. 잠시 후 다시 시도해주세요." |
| 권한 오류 | "403 Forbidden" | "이 작업을 수행할 권한이 없습니다." |
| 유효성 오류 | "Validation failed" | "입력 내용을 확인해주세요." |
| 중복 오류 | "Duplicate entry" | "이미 등록된 {항목}입니다." |

오류 토스트에 [다시 시도] 액션 버튼을 포함할 수 있다. (선택)

```
✕ 저장에 실패했습니다. 잠시 후 다시 시도해주세요.  [다시 시도]
```

## Toast 공통 규칙

| 유형 | 색상 | 지속 시간 | 닫기 버튼 |
|------|------|-----------|-----------|
| 성공 | green | 3초 자동 | 선택 |
| 실패/오류 | red | 5초 자동 또는 수동 | 필수 |
| 경고 | amber | 5초 자동 | 선택 |
| 정보 | blue | 3초 자동 | 선택 |

- 토스트는 동시에 최대 3개까지 쌓이며, 초과 시 가장 오래된 것부터 제거
- 동일한 토스트가 연속 발생하면 중복 표시하지 않고 카운트로 표시 (선택)
- 화면 내 위치는 프로젝트 전체에서 통일 (우측 하단 권장)
