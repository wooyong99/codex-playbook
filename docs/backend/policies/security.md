# Security Policy

백엔드 보안 관련 정책.

---

## 비밀번호

- 비밀번호는 **해시만** 도메인 계층에 진입한다.
- 평문 비밀번호는 API DTO 경계에서 종료한다 (도메인/영속 계층으로 흘러들지 않는다).

---

## 인증 컨텍스트 분리

- 서로 다른 사용자 집합을 대상으로 하는 채널은 **인증 컨텍스트를 분리**한다.
  - 채널별로 사용자 엔티티·자격 증명 저장소·인증 필터를 독립적으로 둔다.
  - 예: 일반 사용자용 채널과 시스템 관리자용 채널은 동일 사용자 테이블·권한 모델을 공유하지 않는다.
- 인증 컨텍스트가 공유되면 권한 상승·세션 오염·감사 추적 단절이 발생할 수 있으므로, 물리적 분리를 기본값으로 한다.

---

## 민감 정보 취급

- `.env`, credentials, 비밀키 등 민감 필드는 **커밋 금지**.
- 민감 정보가 포함된 파일은 `.gitignore`에 사전 등록한다.
